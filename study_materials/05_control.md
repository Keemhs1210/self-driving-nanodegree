# 05. 제어 (Control) — PID & MPC

> 계획된 경로를 따라가도록 **조향·가속·제동** 명령 생성.

## PID 제어 — P9
```
u = -(Kp·e + Ki·∫e + Kd·de/dt)
     비례     적분(정상오차) 미분(진동억제)
```
- **Kp**: 반응 세기(과하면 진동). **Kd**: 감쇠. **Ki**: 누적편향 제거.
- 튜닝: 수동 또는 **Twiddle**(좌표상승 자동최적화).
- 단순·견고하지만 모델을 모름 → 곡선/지연에 한계.

## MPC (모델 예측 제어) — P10
```
미래 N스텝을 운동모델로 예측 → 비용 최소화하는 제어시퀀스 최적화
→ 첫 제어만 적용 → 다음 스텝에 다시 (receding horizon)
```
- 상태 [x,y,ψ,v,cte,eψ], 비용 = cte²+eψ²+(v-v_ref)² + 제어크기·변화량.
- **장점**: 미래를 내다봐 곡선·지연·제약(조향한계)에 강함.
- **단점**: 모델·최적화(Ipopt) 필요, 연산량 큼.

## PID vs MPC
| | PID | MPC |
|--|-----|-----|
| 모델 | 없음 | 운동모델 사용 |
| 미래 | 못 봄 | N스텝 예측 |
| 제약 | 어려움 | 자연스럽게 반영 |
| 비용 | 가벼움 | 무거움 |

## 관련 프로젝트
P9(PID) · P10(MPC)
